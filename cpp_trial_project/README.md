# cpp_trial_project

## プロジェクト概要: テキストRPGバトルシミュレーター

C++のオブジェクト指向プログラミング（OOP）の基本概念（クラス、継承、ポリモーフィズム）を実践するための、シンプルなターン制バトルシミュレーターです。

## 機能要件

*   **ターン制バトル**: プレイヤーと敵が交互に行動します。
*   **コマンド選択**: プレイヤーは「攻撃」「スキル」「防御」などの行動を選択できます。
*   **ステータス管理**: HP（体力）、MP（魔力）、攻撃力などのステータスを持ちます。
*   **勝敗判定**: どちらかのHPが0になった時点で決着がつきます。
*   **ログ出力**: バトルの経過（ダメージ、スキル使用など）をコンソールに表示します。

## クラス構成案 (全6クラス)

このプロジェクトは以下の6つのクラスで構成することを予定しています。

1.  **`Character` (基底クラス)**
    *   全てのキャラクターの共通属性（名前、HP、最大HP、攻撃力）とメソッド（攻撃する、ダメージを受ける）を定義します。
    *   抽象クラスとして実装し、派生クラスで具体的な行動を定義することを想定します。

2.  **`Hero` (Characterから継承)**
    *   プレイヤーが操作するキャラクターです。
    *   独自の属性として `MP` を持ちます。
    *   行動選択のロジック（ユーザー入力の受け付け）を持ちます。

3.  **`Monster` (Characterから継承)**
    *   敵キャラクターです。
    *   シンプルなAI（ランダム行動や特定のパターン）で行動を決定します。

4.  **`Skill`**
    *   スキル（魔法や技）を表現するクラスです。
    *   スキル名、消費MP、威力、効果タイプ（ダメージ/回復）などのデータを持ちます。

5.  **`Dice` (ユーティリティ)**
    *   乱数生成を管理するクラスです。
    *   攻撃の命中判定やダメージの乱数幅、クリティカルヒットの判定などに使用します。

6.  **`GameManager`**
    *   ゲーム全体の進行を管理するクラスです。
    *   `Hero`と`Monster`のインスタンスを保持し、ターン管理、勝敗判定、メインループの制御を行います。

## 作成手順

1.  **環境構築**: C++コンパイラ（g++など）とビルドツール（MakeまたはCMake）の準備。
2.  **基底クラスの実装**: `Character`クラスの定義。
3.  **派生クラスの実装**: `Hero`, `Monster`クラスの実装。
4.  **機能クラスの実装**: `Skill`, `Dice`クラスの実装。
5.  **ゲームロジックの実装**: `GameManager`でバトルフローを構築。
6.  **メイン関数の作成**: `main.cpp`から`GameManager`を呼び出して実行。
